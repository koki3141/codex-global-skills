# Koki PPTX Style

## Slide Logic

- Start from facts, source figures, or numbers, then connect them to one
  necessary conclusion.
- Background slides must not contain the author's self-evaluation such as
  `本研究の社会的意義は...`. State external facts and literature gaps directly.
- Purpose and method slides may use labels such as `本研究の目的は`,
  `本実験では`, and `実験1の結果では` when the label clarifies the setup.
- A primer can define an unfamiliar object. A later background slide must add a
  new role: evidence, limitation, modelling result, side effect, or experiment
  contract. Do not repeat the same definition with different visuals.

## Dense Canvas

- High density is desirable when it carries evidence or logic.
- Preserve numbers, constraints, mechanism labels, comparison conditions,
  equations, evaluation definitions, and causal arrows.
- Remove density created by duplicated wording, repeated definitions, decorative
  boxes, and figure/text saying the same thing.
- Use aligned regions and compact tables to fill space, not decorative cards.

## Typography And Emphasis

- Bottom conclusion: around 24 pt when possible.
- Small evidence headings or local conclusions: around 20 pt.
- Body and claim-supporting evidence: 16 pt or larger.
- Source lines and captions: 8-10 pt is acceptable.
- Use bold and native text underline for causal terms.
- Avoid bullet marks. Show lists through rows, columns, spacing, tables, and
  underlined noun phrases.

## Boxes, Arrows, And Conclusions

- Boxes are structure, not decoration.
- Use boxes for evidence groups that contain distinct facts, numbers, source
  figures, or mechanisms.
- Do not put each causal connector in a box. Use arrows, down-cues, proximity,
  bold text, and underline instead.
- Use one dominant bottom conclusion bar for the integrated takeaway.
- Conclusion bars use dark gray, not pure black. Title headers may be lighter
  gray than conclusion bars.

## Visual Evidence

- Use real source images, source figures, user-provided images, or native
  PowerPoint vector shapes for factual content.
- AI-generated slide images may guide composition, density, or layout balance
  but should not be embedded as evidence.
- For unfamiliar techniques applied repeatedly in the field, include
  field-scale or repeated-layout evidence, not only a close-up.
- Keep a source table with source URL or file, figure number, crop basis,
  reuse/licensing risk, and public-dissemination action.

## Method Figures

- Overall method slide: show the whole contract once.
- Detail slides: show only the local object required for that function.
- Grid figures: every grid line should match the physical cell size when the
  grid represents computation.
- Boundary figures: show numerical values, units, and equations near the
  boundary they define.
- Action figures: show valid candidates, consistent labels, and the minimum
  neighborhood required for the action.

## Verification

- Render slides and inspect full-size previews.
- Check that no body/evidence text fell below 16 pt.
- Check for overlap, clipping, orphan punctuation, awkward line breaks, and
  duplicated slide roles.
- Confirm that cited claims have source links or source-table entries.
- Confirm that the PPTX storage location matches the user's repository or Vault
  policy.
